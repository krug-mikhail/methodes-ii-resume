Mode d'emploi. Ouvre RStudio. Pour chaque question : écris le code R sur le jeu de données indiqué, exécute-le, puis reporte le résultat obtenu dans la case.

Tous les jeux de données (airquality, trees, mtcars, iris, ToothGrowth…) sont déjà intégrés à R — aucun paquet à installer (sauf library(tidyverse) pour les 2 dernières questions). Les résultats numériques sont vérifiés avec une petite tolérance d'arrondi.
1 · Statistiques descriptives
Q1 · code + résultat
Jeu de données airquality. Calcule la moyenne de la colonne Temp. Reporte le résultat (arrondi à 1 décimale).
Code : mean(airquality$Temp) Résultat : 77.88235 (≈ 77.9)
Moyenne arithmétique de la température. Pas de NA dans Temp, donc na.rm est inutile ici.
Q2 · code + résultat
Jeu de données airquality. Calcule la moyenne d'Ozone — attention, cette colonne contient des valeurs manquantes. (arrondi à 1 décimale)
Code : mean(airquality$Ozone, na.rm = TRUE) Résultat : 42.12931 (≈ 42.1)
Sans na.rm = TRUE, R renverrait NA. Le piège classique : oublier cet argument sur une colonne qui a des NA.
Q3 · code + résultat
Jeu de données airquality. Combien de valeurs manquantes y a-t-il dans la colonne Ozone ?
Code : sum(is.na(airquality$Ozone)) Résultat : 37
is.na() renvoie TRUE/FALSE pour chaque valeur ; sum() compte les TRUE (chaque TRUE compte pour 1).
Q4 · code + résultat
Jeu de données trees (mesures d'arbres). Calcule la médiane de la colonne Height.
Code : median(trees$Height) Résultat : 76
La médiane coupe les hauteurs triées en deux moitiés égales.
Q5 · code + résultat
Jeu de données mtcars. Calcule l'écart-type de la colonne mpg. (arrondi à 2 décimales)
Code : sd(mtcars$mpg) Résultat : 6.026948 (≈ 6.03)
sd() = écart-type. Il mesure la dispersion des consommations autour de leur moyenne.
Q6 · code + résultat
Jeu de données iris. Calcule la moyenne de Sepal.Length uniquement pour l'espèce setosa. (arrondi à 2 décimales)
Code (R de base) : mean(iris$Sepal.Length[iris$Species == "setosa"]) ou (dplyr) : iris %>% filter(Species == "setosa") %>% summarize(mean(Sepal.Length)) Résultat : 5.006 (≈ 5.01)
On filtre d'abord les lignes de l'espèce setosa, puis on calcule la moyenne sur ce sous-ensemble.
Q7 · code + résultat
Jeu de données mtcars. Combien de voitures ont une consommation mpg strictement supérieure à 25 ?
Code : sum(mtcars$mpg > 25) ou (dplyr) : mtcars %>% filter(mpg > 25) %>% nrow() Résultat : 6
mtcars$mpg > 25 donne un vecteur TRUE/FALSE ; sum() compte les TRUE.
Q8 · code + résultat
Jeu de données chickwts (poids de poussins selon l'aliment). Combien de types d'aliment (feed) différents y a-t-il ?
Code : length(unique(chickwts$feed)) ou : n_distinct(chickwts$feed) Résultat : 6
unique() liste les valeurs distinctes ; length() les compte. n_distinct() (dplyr) fait les deux d'un coup.
Q9 · code + résultat
Jeu de données swiss (données socio-éco de cantons suisses, 1888). Calcule la médiane de la colonne Agriculture.
Code : median(swiss$Agriculture) Résultat : 54.1
La médiane est plus robuste que la moyenne face aux valeurs extrêmes.
2 · Corrélations
Q10 · code + résultat
Jeu de données trees. Calcule le coefficient de corrélation entre Girth (circonférence) et Volume. (2 décimales)
Code : cor(trees$Girth, trees$Volume) Résultat : 0.9671 (≈ 0.97)
Corrélation très forte et positive : les arbres les plus larges ont un volume plus grand. (Logique — mais corrélation n'est jamais une preuve de causalité.)
Q11 · code + résultat
Jeu de données iris. Calcule la corrélation entre Petal.Length et Petal.Width. (2 décimales)
Code : cor(iris$Petal.Length, iris$Petal.Width) Résultat : 0.9629 (≈ 0.96)
Longueur et largeur des pétales varient quasiment ensemble — corrélation positive très forte.
Q12 · code + résultat
Jeu de données mtcars. Calcule la corrélation entre le poids wt et la consommation mpg. (2 décimales — attention au signe)
Code : cor(mtcars$wt, mtcars$mpg) Résultat : -0.8677 (≈ -0.87)
Corrélation négative forte : plus une voiture est lourde, moins elle parcourt de miles par gallon. Le signe est essentiel.
Q13 · code + résultat
Jeu de données cars (vitesse et distance de freinage). Calcule la corrélation entre speed et dist. (2 décimales)
Code : cor(cars$speed, cars$dist) Résultat : 0.8069 (≈ 0.81)
Corrélation positive forte : plus la vitesse est élevée, plus la distance de freinage est longue.
3 · Test t & ANOVA
Q14 · code + résultat
Jeu de données ToothGrowth. Réalise un test t comparant la longueur len selon le supplément supp (OJ vs VC). Reporte la p-value (2 décimales).
Code : t.test(len ~ supp, data = ToothGrowth) Résultat : p-value = 0.06063 (≈ 0.06)
Le test t avec la syntaxe variable ~ groupe compare les deux moyennes. La p-value se lit dans la sortie.
Q15 · interprétation
D'après la p-value obtenue à la Q14, la différence entre OJ et VC est-elle significative au seuil de 5 % ? (oui / non)
Réponse : Non.
p ≈ 0.061 est légèrement supérieur à 0.05 → on ne rejette pas H0 : la différence n'est pas significative (de justesse !).
Q16 · code + résultat
Jeu de données PlantGrowth. Réalise une ANOVA du poids weight selon le groupe group. Reporte la valeur F (F value, 1 décimale).
Code : summary(aov(weight ~ group, data = PlantGrowth)) Résultat : F value = 4.846 (≈ 4.85)
La valeur F se lit dans la ligne group de la sortie de summary(aov(...)).
Q17 · interprétation
L'ANOVA de la Q16 donne une p-value Pr(>F) = 0.0159. Au moins un groupe a-t-il une moyenne significativement différente ? (oui / non)
Réponse : Oui.
p = 0.0159 < 0.05 → on rejette H0. Au moins un groupe diffère (un test de Tukey dirait lequel).
4 · Régression linéaire
Q18 · code + résultat
Jeu de données mtcars. Ajuste la régression lm(mpg ~ hp) et reporte le coefficient (Estimate) de hp (3 décimales — attention au signe).
Code : summary(lm(mpg ~ hp, data = mtcars)) Résultat : coefficient de hp = -0.06823 (≈ -0.068)
Coefficient négatif : chaque cheval-vapeur en plus fait baisser mpg de 0.068 en moyenne.
Q19 · code + résultat
Même régression lm(mpg ~ hp) sur mtcars. Reporte le Multiple R-squared (2 décimales).
Code : summary(lm(mpg ~ hp, data = mtcars)) Résultat : Multiple R-squared = 0.6024 (≈ 0.60)
La puissance hp explique environ 60 % de la variation de la consommation. Les 40 % restants viennent d'autres facteurs (poids, etc.).
Q20 · code + résultat
Jeu de données cars. Ajuste lm(dist ~ speed) et reporte le coefficient de speed (2 décimales).
Code : summary(lm(dist ~ speed, data = cars)) Résultat : coefficient de speed = 3.9324 (≈ 3.93)
Pente positive : chaque unité de vitesse en plus rallonge la distance de freinage de ≈ 3.93 en moyenne.
5 · Regrouper & comparer
Q21 · code + résultat
Jeu de données chickwts. En regroupant par feed, quel aliment donne le poids moyen le plus élevé ? (reporte le nom de l'aliment ; nécessite library(tidyverse))
Code : chickwts %>% group_by(feed) %>% summarize(moy = mean(weight)) %>% arrange(desc(moy)) Résultat : sunflower (≈ 328.9 g de moyenne)
Schéma group_by + summarize + arrange(desc()) : on calcule la moyenne par groupe puis on trie pour voir le plus élevé. sunflower devance de peu casein.
Q22 · code + résultat
Jeu de données InsectSprays (insecticides). En regroupant par spray, quel spray a le nombre moyen d'insectes le plus FAIBLE (le plus efficace) ? (reporte la lettre du spray)
Code : InsectSprays %>% group_by(spray) %>% summarize(moy = mean(count)) %>% arrange(moy) Résultat : spray C (≈ 2.08 insectes en moyenne)
Avec arrange(moy) (ordre croissant), la 1re ligne est le plus petit : le spray C est le plus efficace, loin devant A, B et F.
