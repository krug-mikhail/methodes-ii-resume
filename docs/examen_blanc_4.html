Examen blanc 4 — Synthèse corsée

Tous les formats réunis · niveau le plus exigeant · 28 questions

Le plus dur des quatre. Cet examen mélange les 4 formats : 🔴 Exécute & reporte (lance le code dans RStudio, donne le résultat) · 🟣 Code à écrire (verifié sur la structure) · 🔵 QCM (avec des pièges !) · 🟡 Interprétation.

Garde RStudio ouvert. Les jeux de données sont intégrés à R (sauf library(tidyverse) pour les chaînes dplyr). Lis bien les énoncés — plusieurs questions contiennent des pièges classiques.
1 · Descriptives & manipulation
Exécute & reporte · Q1
Jeu de données airquality. Calcule la corrélation entre Ozone et Temp — attention, Ozone a des NA, utilise les observations complètes. (2 décimales)
Code : cor(airquality$Ozone, airquality$Temp, use = "complete.obs") Résultat : 0.6984 (≈ 0.70)
Avec des NA, cor() renvoie NA par défaut. L'argument use = "complete.obs" ne garde que les lignes sans valeur manquante.
Code à écrire · Q2
Écris une chaîne dplyr qui : prend airquality, garde uniquement les lignes du mois 7 (Month == 7), puis calcule la moyenne d'Ozone (en ignorant les NA).
Solution : airquality %>% filter(Month == 7) %>% summarize(moy = mean(Ozone, na.rm = TRUE))
Schéma filter → summarize : on restreint d'abord les lignes, puis on résume. na.rm = TRUE reste indispensable.
Exécute & reporte · Q3
Exécute le code de la Q2 : quelle est la moyenne d'Ozone au mois de juillet (Month 7) ? (1 décimale)
Résultat : 59.11538 (≈ 59.1)
Juillet est un mois chaud → taux d'ozone élevé. À comparer : en mai (Month 5), la moyenne n'est que de ≈ 23.6.
Exécute & reporte · Q4
Jeu de données airquality. Combien de lignes sont complètes (aucune valeur manquante sur aucune colonne) ?
Code : sum(complete.cases(airquality)) ou (dplyr) : airquality %>% drop_na() %>% nrow() Résultat : 111 # sur 153 lignes
complete.cases() renvoie TRUE pour les lignes sans aucun NA. 42 lignes sont donc incomplètes.
QCM · Q5
La commande mean(airquality$Ozone) renvoie NA. Pourquoi ?
  • Le jeu de données airquality n'existe pas
  • La colonne Ozone contient des NA — sans na.rm = TRUE, la moyenne est NA
  • Ozone est une variable textuelle
  • Il faut d'abord charger un paquet
Dès qu'un seul NA est présent, mean() (comme sd(), median()…) renvoie NA. Solution : mean(airquality$Ozone, na.rm = TRUE).
Exécute & reporte · Q6
Jeu de données mtcars. Calcule la moyenne de mpg uniquement pour les voitures à 4 cylindres (cyl == 4). (2 décimales)
Code : mean(mtcars$mpg[mtcars$cyl == 4]) Résultat : 26.66364 (≈ 26.66)
Les 4-cylindres consomment bien moins : leur moyenne (≈ 26.7) est presque le double de celle des 8-cylindres (≈ 15.1).
2 · Test t
Code à écrire · Q7
Jeu de données PlantGrowth (colonnes weight et group). Écris le code d'un test t comparant le weight des groupes ctrl et trt2 uniquement.
Solution (au choix) : # Approche 1 : deux vecteurs ctrl <- PlantGrowth$weight[PlantGrowth$group == "ctrl"] trt2 <- PlantGrowth$weight[PlantGrowth$group == "trt2"] t.test(ctrl, trt2) # Approche 2 : filtrer puis formule PlantGrowth %>% filter(group %in% c("ctrl", "trt2")) %>% t.test(weight ~ group, data = .)
Le test t ne compare que 2 groupes : il faut donc écarter trt1, soit en extrayant 2 vecteurs, soit en filtrant.
Exécute & reporte · Q8
Exécute ce test t (ctrl vs trt2 de PlantGrowth). Reporte la p-value (3 décimales).
Résultat : p-value = 0.0479 (≈ 0.048)
p ≈ 0.048 — juste en dessous de 0.05.
Interprétation · Q9
D'après la Q8 (p ≈ 0.048), la différence ctrl vs trt2 est-elle significative au seuil de 5 % ? (oui / non)
Réponse : Oui — mais de justesse.
0.048 < 0.05 → on rejette H0. C'est significatif, mais « limite » : avec un seuil un peu plus strict ou un autre échantillon, la conclusion pourrait basculer.
QCM · Q10 — piège
Test A : p = 0.04. Test B : p = 0.0001. Que peut-on conclure ?
  • Le test B révèle un effet beaucoup plus grand que le test A
  • Les deux sont significatifs ; mais la p-value ne mesure PAS la taille de l'effet
  • Le test A n'est pas significatif
  • Le test B est forcément plus important en pratique
Piège classique. Une p-value plus petite = preuve plus forte contre H0, pas un effet plus grand. La taille de l'effet se mesure autrement (différence de moyennes, coefficient, r…).
QCM · Q11
Dans quel cas utilise-t-on un test t apparié (paired = TRUE) ?
  • Quand on compare 3 groupes ou plus
  • Quand les deux mesures portent sur les mêmes individus (ex. avant / après)
  • Quand les variances des groupes sont égales
  • Quand l'échantillon dépasse 30 observations
Le test apparié sert quand chaque individu est mesuré deux fois (avant/après un traitement). Sinon, on utilise le test t à deux échantillons indépendants.
3 · ANOVA
Exécute & reporte · Q12
Jeu de données iris. Réalise une ANOVA de Sepal.Length selon Species. Reporte la valeur F (arrondie à l'entier).
Code : summary(aov(Sepal.Length ~ Species, data = iris)) Résultat : F value = 119.3 (≈ 119)
F très élevé → forte séparation des espèces sur la longueur du sépale.
Code à écrire · Q13
Jeu de données ToothGrowth. Écris le code d'une ANOVA de len selon la dose — la dose est numérique, il faut la traiter comme un facteur.
Solution : summary(aov(len ~ factor(dose), data = ToothGrowth))
dose contient les valeurs 0.5, 1, 2 (numériques). Pour une ANOVA, il faut la transformer en facteur (3 groupes) avec factor(dose).
Exécute & reporte · Q14
Exécute l'ANOVA de la Q13. Reporte la valeur F (arrondie à l'entier).
Résultat : F value = 67.42 (≈ 67)
F élevé, p-value minuscule → la dose a un effet très net sur la croissance dentaire.
QCM · Q15
Une ANOVA sur 3 groupes est significative. Quelle est l'étape logique suivante ?
  • Refaire exactement la même ANOVA
  • Un test post-hoc (TukeyHSD) pour identifier quelles paires de groupes diffèrent
  • Une régression linéaire à la place
  • Rien, l'analyse est terminée
L'ANOVA dit qu'au moins un groupe diffère, sans préciser lequel. Le TukeyHSD compare les paires pour le savoir.
Interprétation · Q16
L'ANOVA len ~ factor(dose) a une p-value Pr(>F) proche de 0. La dose a-t-elle un effet significatif sur len ? (oui / non)
Réponse : Oui.
p ≈ 0 ≪ 0.05 → on rejette H0 : au moins une dose produit une longueur moyenne différente.
4 · Régression linéaire
Exécute & reporte · Q17
Jeu de données mtcars. Ajuste la régression multiple lm(mpg ~ wt + hp). Reporte le coefficient de hp (3 décimales — attention au signe).
Code : summary(lm(mpg ~ wt + hp, data = mtcars)) Résultat : coefficient de hp = -0.03177 (≈ -0.032)
À poids constant, chaque cheval-vapeur supplémentaire fait baisser mpg de 0.032 en moyenne.
Exécute & reporte · Q18
Même modèle lm(mpg ~ wt + hp). Reporte le Multiple R-squared (2 décimales).
Résultat : Multiple R-squared = 0.8268 (≈ 0.83)
Le modèle à 2 variables explique ≈ 83 % de la variance de mpg — mieux que wt seul (75 %).
Interprétation · Q19
Dans ce modèle, le coefficient de hp a une p-value Pr(>|t|) = 0.0015. La puissance hp est-elle un prédicteur significatif ? (oui / non)
Réponse : Oui.
0.0015 < 0.05 → le coefficient de hp est significativement différent de 0 : hp apporte de l'information même une fois wt pris en compte.
QCM · Q20
Toujours dans lm(mpg ~ wt + hp), le coefficient de wt vaut -3.88. Interprétation correcte ?
  • mpg vaut toujours -3.88
  • À puissance (hp) constante, +1 unité de poids fait baisser mpg de 3.88 en moyenne
  • wt explique 3.88 % de la variance
  • Le poids n'a pas d'effet significatif
En régression multiple, chaque coefficient mesure l'effet de sa variable « toutes choses égales par ailleurs » — ici, à hp maintenu constant.
Exécute & reporte · Q21
Jeu de données swiss. Ajuste lm(Fertility ~ Education). Reporte le coefficient de Education (2 décimales — attention au signe).
Code : summary(lm(Fertility ~ Education, data = swiss)) Résultat : coefficient de Education = -0.8624 (≈ -0.86)
Coefficient négatif : dans ces données de 1888, plus l'éducation est élevée, plus la fécondité est basse.
Code à écrire · Q22
Jeu de données trees. Écris le code qui ajuste la régression de Volume par Girth, et qui en affiche le résumé.
Solution : modele <- lm(Volume ~ Girth, data = trees) summary(modele) # ou en une ligne : summary(lm(Volume ~ Girth, data = trees))
Ici R² ≈ 0.94 : la circonférence explique à elle seule 94 % de la variation du volume.
QCM · Q23
Modèle simple : R² = 0.60. On ajoute 3 variables : R² = 0.62. Sur quoi se baser pour juger si l'ajout en valait la peine ?
  • Le R² simple : il a augmenté, donc on garde le gros modèle
  • Le R² ajusté, qui pénalise les variables superflues
  • La p-value de l'intercept
  • Le nombre de lignes du jeu de données
Le Multiple R-squared monte toujours quand on ajoute des variables, même inutiles. Le Adjusted R-squared (et le critère AIC) corrigent ce biais.
5 · Interprétation & pièges
QCM · Q24 — piège
Une section Résultats écrit : « le test n'étant pas significatif (p = 0.21), nous avons prouvé qu'il n'y a aucun effet. » Quelle est l'erreur ?
  • Aucune erreur, la phrase est correcte
  • « Non significatif » ne prouve pas l'absence d'effet — ça peut être un manque de puissance
  • p = 0.21 est en réalité significatif
  • Il fallait écrire « p = 21 % »
Un test non significatif = « pas assez de preuves », et non « preuve qu'il n'y a rien ». L'effet peut exister mais l'échantillon être trop petit pour le détecter.
Interprétation · Q25
Sur swiss, on trouve cor(Fertility, Education) = -0.66. Le lien entre fécondité et éducation est-il positif ou négatif ? (un mot)
Réponse : Négatif.
Le signe de r donne la direction : ici négatif → quand l'éducation augmente, la fécondité tend à baisser. La valeur 0.66 indique une force moyenne-à-forte.
QCM · Q26 — piège
Chez des enfants, on mesure r = 0.97 entre la pointure et le niveau en lecture. Conclusion correcte ?
  • Avoir de grands pieds rend meilleur en lecture
  • C'est une corrélation, très probablement due à une variable cachée (l'âge) — pas une causalité
  • Savoir lire fait grandir les pieds
  • Une corrélation de 0.97 est impossible
Cas d'école de variable cachée : l'âge fait grandir les pieds et améliore la lecture. La corrélation est réelle, mais il n'y a aucun lien de cause à effet direct.
QCM · Q27
Quelle formulation rapporte correctement un test t dans une section Résultats ?
  • « Le test était clairement significatif. »
  • « Les groupes diffèrent significativement, t(45) = 3.12, p < .001. »
  • « La statistique t était grande, donc c'est bon. »
  • « p valait beaucoup, donc significatif. »
Notation standard (style APA) : la statistique t avec ses degrés de liberté entre parenthèses, puis la p-value — des chiffres précis, pas du vague.
Exécute & reporte · Q28
Jeu de données iris. En regroupant par Species, quelle espèce a la plus grande largeur de sépale moyenne (Sepal.Width) ? (nom de l'espèce ; library(tidyverse))
Code : iris %>% group_by(Species) %>% summarize(moy = mean(Sepal.Width)) %>% arrange(desc(moy)) Résultat : setosa (≈ 3.43 — devant virginica 2.97 et versicolor 2.77)
Contre-intuitif : setosa est la plus petite espèce en longueur de pétale, mais celle qui a les sépales les plus larges.
